# Project Constitution

## Dependency Constraints

- **Runtime dependencies**: Limited to 5 transitive dependencies (crates)
- **Dev-dependencies**: Limited to an additional 5 transitive dependencies
- **Never** use dependencies with procedural macros or heavy build scripts
- **Prefer minimal dependency trees**: Choose dependencies with no or few transitive dependencies to preserve room for adding new dependencies in the future

## Code Style

- Write comments to explain the **why** of the code, especially for complex or non-obvious logic
- Focus on intent and reasoning, not just describing what the code does
- Never leave trailing whitespace at the end of lines
- Run `cargo clippy` before committing and address any warnings

## Testing

- In Rust, unit tests usually go in a `tests` sub-module of the module we want to test

## Project Structure

- Create the Rust project at the repository root (not in a subdirectory)
- Place all markdown files generated by agents (specs, tasks, reports, etc.) in a subdirectory, not at the project root

## Git Workflow

- All changes should be done in a feature branch
- When fixing a bug:
  1. First create a regression test and commit it
  2. Then fix the bug in a separate commit
