# Project Constitution

## Dependency Constraints

- Limit to **5 transitive dependencies** (crates)
- Prefer dependencies with no or few dependencies to keep room for adding new dependencies in the future
- dev-dependencies should be limited to an additional **5 transitive dependencies**
- **Never** use dependencies with procedural macros or heavy build scripts

## Code Style

- Write comments to explain the **why** of the code, especially for complex or non-obvious logic

## Workflow

- All changes should be done in a **feature branch**
- When fixing a bug:
  1. First create a regression test and commit it
  2. Then fix the bug in a new commit

## Project Structure

- Create the Rust project at the **root** instead of a subdirectory
- Place markdown files generated by agents (specs, tasks, report, etc.) in a **subdirectory** instead of at the project root

## Tooling

- Use **Serena** to index code and search for relevant documentation
- **Never** run code formatters (e.g., `cargo fmt`, `rustfmt`, `prettier`, etc.)
